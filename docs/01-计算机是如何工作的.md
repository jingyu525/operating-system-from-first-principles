## 第一性原理

计算机最核心的三样东西：

```text
CPU ←→ 内存 ←→ IO 设备（磁盘、网卡等）
```

这条总线连接一切：

```
     ┌──────┐      数据       ┌──────┐
     │ CPU  │ ←───────────→  │ 内存  │
     └──┬───┘                 └──┬───┘
        │                        │
        │      控制总线           │
        │ ←─────────────────────→│
        │                        │
        │      地址总线           │
        │ ←─────────────────────→│
        │                 ┌──────┴──────┐
        └─────────────────┤  IO 控制器   │
                          └──────┬──────┘
                                 │
                    ┌────────────┼────────────┐
                    │            │            │
                 磁盘控制器    网卡         显卡
```

## 推导过程

### CPU 的视角

CPU 只做一件事：

```text
while (true) {
    取指令(PC 指向的内存地址);
    执行指令;
    PC++;  // 程序计数器 +1
}
```

这就是**冯·诺依曼架构**的核心：**指令和数据都存在同一块内存里**。

### 内存的视角

内存就是一大块字节数组：

```text
地址    内容
0x0000: [指令 1]
0x0004: [指令 2]
0x0008: [数据 A]
0x000C: [数据 B]
...
```

CPU 通过地址总线告诉内存"我要读地址 0x0008 的内容"，内存返回数据。

### 缓存（Cache）

```text
CPU 速度：~0.3ns / 周期
内存速度：~100ns / 次访问

差距：~300 倍
```

所以有了缓存层级：

```text
CPU 寄存器 (最快，~0 cycles)
  │
L1 Cache (~4 cycles)
  │
L2 Cache (~12 cycles)
  │
L3 Cache (~40 cycles)
  │
内存 (~200 cycles)
```

**局部性原理**：程序倾向于访问附近的数据和指令，所以缓存命中率很高。

## 核心概念

| 概念 | 本质 |
|------|------|
| **冯·诺依曼架构** | 指令和数据存在同一内存中 |
| **程序计数器 (PC)** | CPU 下一条要执行指令的地址 |
| **寄存器** | CPU 内部最快的存储单元 |
| **Cache** | 弥补 CPU 和内存速度差距的缓冲区 |
| **总线** | CPU、内存、IO 之间的数据高速公路 |
| **中断** | 硬件通知 CPU 的方式（如网卡收到数据） |

## Linux 是怎么实现的？

Linux 内核启动过程：

```text
BIOS/UEFI → Bootloader (GRUB) → Kernel → init (systemd) → Login
```

关键：内核本身也是一个程序，但它运行在**最高权限级别**（Ring 0），可以访问所有硬件。

## Go 是怎么利用它的？

```go
func main() {
    x := 42        // x 在栈上（CPU cache 友好）
    y := new(int)  // y 在堆上（可能在内存中）
    *y = 100
}
```

Go 编译器做大量优化，让变量尽量留在寄存器或 Cache 中：

```bash
# 查看编译优化
go build -gcflags="-m" main.go
```

## 常见面试题

**Q: 为什么需要 Cache？直接让 CPU 访问内存不行吗？**

A: CPU 比内存快 ~300 倍。没有 Cache，CPU 大部分时间都在等内存，这就是"内存墙"。Cache 利用**局部性原理**，把常用数据放在离 CPU 近的地方。

**Q: 寄存器和内存有什么区别？**

A: 寄存器在 CPU 内部，访问延迟 ~0 cycles；内存在 CPU 外部，访问延迟 ~200 cycles。编译器尽力把热变量放在寄存器里。

## 实战

```bash
# 查看 CPU 信息
lscpu

# 查看 Cache 大小
lscpu | grep -i cache

# 查看内存信息
free -h

# 查看程序 Cache 命中率
perf stat -e cache-references,cache-misses ls
```

---

## 总结

> 计算机的核心就是 CPU 不断地从内存取指令、执行指令，Cache 是弥补 CPU 和内存速度差距的关键。
