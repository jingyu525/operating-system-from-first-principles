# 00 — 为什么需要操作系统

---

## 这一章回答什么问题？

> 没有操作系统的计算机是什么样子？操作系统到底解决了什么问题？

---

## 第一性原理

计算机硬件只能做最简单的事：

```text
CPU：取指令 → 执行指令 → 取指令 → 执行指令 → ...
内存：读字节 / 写字节
磁盘：读扇区 / 写扇区
网卡：发数据包 / 收数据包
```

如果直接操作硬件：

```text
你的程序
  │
  ├── 直接操作 CPU
  ├── 直接操作内存地址
  ├── 直接操作磁盘扇区
  └── 直接操作网卡寄存器
```

**问题是：**

1. 每个程序都要自己写磁盘驱动？
2. 两个程序同时访问磁盘怎么办？
3. 一个程序崩溃，怎么不影响其他程序？
4. 怎么在只有一个 CPU 的情况下运行多个程序？

---

## 推导过程

```
没有 OS 的世界：
┌─────────────────────────────────┐
│ 程序 A                          │
│ - 自己管理磁盘                   │
│ - 自己管理内存                   │
│ - 自己处理网卡                   │
├─────────────────────────────────┤
│ 程序 B                          │
│ - 自己管理磁盘                   │
│ - 自己管理内存                   │
│ - 自己处理网卡                   │
└─────────────────────────────────┘
    ▲ 大量重复 + 互相冲突

有了 OS 的世界：
┌─────────────────────────────────┐
│ 程序 A     程序 B     程序 C    │
├─────────────────────────────────┤
│         操作系统 (OS)            │
│  - CPU 管理                     │
│  - 内存管理                     │
│  - 文件系统                     │
│  - 网络栈                       │
├─────────────────────────────────┤
│        硬件 (CPU/内存/磁盘/网卡) │
└─────────────────────────────────┘
```

一句总结：

> **操作系统本质是硬件资源管理器。** 它把杂乱无章的硬件抽象成整洁的接口，让程序之间和平共处。

---

## 核心概念

| 概念 | 本质 |
|------|------|
| **抽象** | 把复杂的硬件操作封装成简单接口（`read()` / `write()`) |
| **隔离** | 程序 A 崩溃不影响程序 B |
| **共享** | 多个程序共用同一个 CPU、内存、磁盘 |
| **权限** | 不能让普通程序随便读写磁盘任意位置 |

## Linux 是怎么实现的？

Linux 内核的两大职责：

```text
         Linux Kernel
              │
  ┌───────────┴───────────┐
  │                       │
进程管理                 资源抽象
  │                       │
  ├── 进程/线程           ├── 虚拟内存
  ├── 调度器              ├── 文件系统 (VFS)
  ├── 信号                ├── 网络栈
  └── IPC                 └── 设备驱动
```

## Go 是怎么利用它的？

Go 程序看似"直接运行"，实际上：

```go
package main

import "os"

func main() {
    f, _ := os.Open("test.txt")  // → 系统调用 → 内核 → 文件系统 → 磁盘驱动 → 磁盘
    f.Read(buf)                  // → 系统调用 → 内核 → Page Cache
    f.Close()
}
```

Go 的 Runtime 帮你管理 goroutine，但最后的最后，一切都经过 OS。

---

## 常见面试题

**Q: 为什么需要操作系统？直接操作硬件不行吗？**

A: 直接操作硬件 = 每个程序都要重新发明轮子（磁盘驱动、内存管理、进程调度），而且多个程序会互相冲突。操作系统提供**抽象 + 隔离 + 共享**，让程序可以简单安全地运行。

## 实战

```bash
# 看看你的系统跑了多少进程
ps aux | wc -l

# 看看内核版本

# 看看系统调用

```

---

## 总结

> 操作系统 = 硬件资源管理器，提供抽象、隔离、共享和安全。

## 与后端开发的联系

```text
你的 Go 服务
  │
  ▼
net/http → socket → 内核网络栈 → 网卡
  │
  ▼
数据库连接 → socket → 内核 → MySQL 进程
  │
  ▼
日志 → write() → 内核 → Page Cache → 磁盘
```

你写的每一行代码，最终都在和 OS 打交道。
