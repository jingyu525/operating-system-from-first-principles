# 02 — 程序为什么能够运行

---

## 这一章回答什么问题？

> 你写的 `main.go` 是怎么变成一个真正在 CPU 上运行的进程的？

---

## 第一性原理

```text
CPU 只能执行机器码。
CPU 只认识二进制。
```

所以：

```text
源码（人类可读）
  ↓
编译器
  ↓
机器码（CPU 可执行）
  ↓
操作系统加载
  ↓
进程
```

---

## 推导过程

### 第一步：编译

```text
main.go
  │
  ▼
词法分析 → Token 流
  │
  ▼
语法分析 → AST（抽象语法树）
  │
  ▼
类型检查
  │
  ▼
中间代码生成 → SSA（静态单赋值形式）
  │
  ▼
优化
  │
  ▼
机器码生成 → 目标文件 (.o)
```

### 第二步：链接

```text
main.o + fmt.o + runtime.o + ...
  │
  ▼
链接器
  │
  ▼
ELF 可执行文件
```

### 第三步：加载

```text
终端输入 ./myapp
  │
  ▼
Shell 调用 execve() 系统调用
  │
  ▼
Linux Loader 解析 ELF
  │
  ▼
分配虚拟地址空间
  │
  ▼
加载代码段、数据段到内存
  │
  ▼
设置栈、堆
  │
  ▼
跳转到 main() 入口
  │
  ▼
进程开始运行！
```

## ELF 文件结构

```text
ELF Header          ← 魔数、架构、入口地址
  │
Program Headers     ← 告诉 Loader 怎么加载（运行时需要）
  │
  ├── .text         ← 代码段（只读）
  ├── .rodata       ← 只读数据（常量字符串）
  ├── .data         ← 已初始化全局变量
  ├── .bss          ← 未初始化全局变量（不占磁盘空间！）
  └── ...
  │
Section Headers     ← 给链接器和调试器看的
```

## 进程的地址空间

```text
高地址
┌─────────────────┐
│     Stack       │ ← 局部变量，函数调用，向下增长
│       ↓         │
│                 │
│       ↑         │
│     Heap        │ ← 动态分配 (malloc / new)，向上增长
├─────────────────┤
│    .bss         │ ← 未初始化全局变量
├─────────────────┤
│    .data        │ ← 已初始化全局变量
├─────────────────┤
│    .text        │ ← 代码
├─────────────────┤
│   Reserved      │ ← NULL 指针区域（不可访问）
└─────────────────┘
低地址
```

## 核心概念

| 概念 | 本质 |
|------|------|
| **编译** | 源码 → 机器码 |
| **链接** | 多个 .o 拼成可执行文件 |
| **ELF** | Linux 可执行文件格式 |
| **加载** | 把 ELF 内容映射到内存 |
| **地址空间** | 进程看到的内存布局 |
| **静态链接 vs 动态链接** | 编译时 vs 运行时绑定库 |

## Go 是怎么利用它的？

Go 默认**静态链接**：

```bash
go build main.go
file main          # ELF 64-bit executable, statically linked
ldd main           # not a dynamic executable
```

好处：一个二进制文件跑遍所有 Linux，不需要安装依赖。

代价：二进制文件较大（因为把 runtime 和标准库都打包进去了）。

```go
// Go 的启动流程（简化）
_rt0_amd64_linux()     // 汇编入口
  → runtime.rt0_go()
    → runtime.args()   // 解析命令行参数
    → runtime.osinit() // 初始化 OS 相关（CPU 数量等）
    → runtime.schedinit() // 初始化调度器
    → runtime.newproc()   // 创建 main goroutine
    → runtime.mstart()    // 开始调度
      → main.main()       // 终于执行你的 main 函数！
```

## 常见面试题

**Q: 静态链接和动态链接的区别？**

A:
- 静态链接：编译时把所有库打包进可执行文件，文件大但独立运行
- 动态链接：运行时加载 `.so` 文件，文件小但依赖系统环境

Go 默认静态链接 → 一个二进制打天下。

**Q: 为什么 `.bss` 段不占磁盘空间？**

A: `.bss` 是全零的未初始化变量，ELF 只记录"需要多少字节"，加载时操作系统分配即可，不需要在磁盘上存零。

## 实战

```bash
# 查看 ELF 头
readelf -h myapp

# 查看段信息
readelf -l myapp

# 查看节信息
readelf -S myapp

# 查看符号表
nm myapp | head -20

# 查看动态链接库
ldd /bin/ls

# 查看 Go 程序是否静态链接
go build main.go && ldd main
```

---

## 总结

> 编译把源码翻译成机器码，链接把多个目标文件拼成 ELF，加载器把 ELF 映射到虚拟地址空间——然后 CPU 开始执行 `main()`。
