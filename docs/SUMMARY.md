# SUMMARY — 操作系统从第一性原理出发

## 全书的逻辑主线

```
为什么需要操作系统？
  │
  ▼
计算机是怎么工作的？
  │
  ▼
程序为什么能运行？（编译 → 链接 → 加载）
  │
  ├── CPU 管理 ──→ 进程 → 线程 → 调度 → 上下文切换
  │
  ├── Memory 管理 ──→ 虚拟内存 → (mmap/brk)
  │
  ├── IO 管理 ──→ 文件系统 → 网络 IO (Socket/epoll)
  │
  ▼
系统调用（用户态 ↔ 内核态的桥梁）
  │
  ▼
Go Runtime 与 OS 的关系
  │
  ▼
性能分析（把上面所有知识用起来）
  │
  ▼
一行 Go 代码的完整旅程（串联全书）
```

---

## 关键概念速查

| 概念 | 一句话本质 | 章节 |
|------|-----------|------|
| OS | 硬件资源管理器 | 00 |
| CPU | 一次只能执行一条指令 | 01 |
| 进程 | CPU 的抽象 | 03 |
| 线程 | CPU 调度的最小单位 | 04 |
| Scheduler | CPU 资源分配器 | 05 |
| Context Switch | 保存/恢复 CPU 现场 | 06 |
| 虚拟内存 | 每个进程的地址空间幻觉 | 07 |
| 系统调用 | 用户程序进入内核的门 | 08 |
| 文件系统 | 磁盘空间的组织方式 | 09 |
| epoll | 事件驱动的 IO 多路复用 | 10 |
| GMP | Go 的用户态调度器 | 11 |

---

## 全书设计哲学

> 不是讲操作系统，而是讲现代互联网系统为什么能够运行。
